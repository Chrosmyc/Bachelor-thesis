# Secondary Risk Created
Replacement cost and integration effort.

# Applicability Conditions
Third-party packaged software with limited documentation or weak vendor support.

# Observed Outcome
Package replacement mitigates maintenance risk when third-party legacy package functionality is poorly documented or no longer well supported.

# Reasoning Notes
Package replacement for poorly documented or unsupported components (M86) targets the root cause of Legacy-Modern Coexistence Failure (R96): Lack of support for hybrid environments.

# Standardized Relationship Entry
IF Replace packaged or unsupported components when reuse would lead to long-term maintenance problems or when vendor infrastructure is hardly updated and maintained THEN the risk of legacy-modern coexistence failure (R96) is prevented.
