# Secondary Risk Created
Reduced direct control over outsourced feature development.

# Applicability Conditions
Low-priority features such as card authorization and card payments.

# Observed Outcome
Controlled outsourcing mitigates focus and cost risks by limiting external development to low-priority features under strict strategic guidelines.

# Reasoning Notes
Controlled outsourcing for low-priority features (M87) targets the root cause of Uncontrolled handling of migration changes (R90): The migration process contains many dependencies and changing business/architecture priorities.

# Standardized Relationship Entry
IF Outsource only low-business-value features when outsourcing supports strategic objectives, follows outsourcing guidelines, uses strategic partners, and no other option is viable THEN the risk of uncontrolled handling of migration changes (R90) is prevented.
