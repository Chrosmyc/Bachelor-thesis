# Secondary Risk Created
A more complex routing mechanism may be required depending on the system.

# Applicability Conditions
Systems where consumer traffic can be routed or mediated.

# Observed Outcome
Introduce an indirection layer to route consumers between legacy and new components during incremental migration.

# Reasoning Notes
Introduce an indirection layer (M141) targets the root cause of Prolonged dual-system transition (R135): Incremental replacement intentionally keeps both systems active during migration.

# Standardized Relationship Entry
IF Add a software seam, switch, routing mechanism, or API gateway that allows new components to be introduced transparently to consumers THEN the risk of prolonged dual-system transition (R135) is prevented.
