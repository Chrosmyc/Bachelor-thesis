# Description
The architecture of the legacy capability and the target SOA environment may not fit together cleanly.

# Root Cause
Differences between existing architecture and the SOA interaction, deployment, or communication model.

# Trigger Event
Mapping legacy components to services without resolving architectural differences.

# Impact
Incompatibility, difficult integration, redesign, or unstable service behavior.

# Linked Risks
- R77

# Stakeholders

- Architects
- developers
- service consumers

# Contextual Constraints

- Applies when existing component architecture differs from the target SOA environment's assumptions.


# Detection Source
Architecture Diagram / Interview

# Standardized Risk Entry
Architectural mismatch between existing capability and target SOA can threaten feasibility and increase adaptation work.

# Key Quote for Prompting
> "Architectural mismatch"

# Logic Pattern
IF legacy architecture and target SOA assumptions do not align AND migration proceeds THEN integration and redesign risk arise.
