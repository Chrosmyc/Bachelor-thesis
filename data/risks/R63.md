# Description
The migration may define inappropriate microservice candidates, responsibilities, or sizes; the paper states that defining responsibility and size is crucial for the migration process.

# Root Cause
Poor decomposition of the legacy system into units to be transformed into microservices; insufficient consideration of coupling, cohesion, features, reuse, dependencies, data flow, and technologies.

# Trigger Event
Identifying microservice candidates without adequate decomposition criteria or system knowledge.

# Impact
The paper does not specify concrete consequences, but inappropriate candidates can undermine the migration process and architecture definition.

# Linked Risks
- R65; R67

# Stakeholders

- Engineers responsible for migration; development teams

# Contextual Constraints

- Applies during identification of microservice candidates.


# Detection Source
Source Code / Features / Dependencies / Data Flow / Technology List

# Standardized Risk Entry
If microservice candidates are identified without carefully defining service responsibility and size, the migration process may be based on unsuitable decomposition units.

# Key Quote for Prompting
> "Based on the importance of defining the responsibility and size of a microservice, this step is a crucial for the migration process."

# Logic Pattern
IF decomposition criteria are weak AND microservice candidates are selected THEN unsuitable microservice boundaries may be produced.
