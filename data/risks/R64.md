# Description
Keeping a central shared database in a microservice context can lead to data coupling among microservices.

# Root Cause
Microservices share access to a central database instead of each microservice managing its own data through API-based access.

# Trigger Event
Engineers keep a central shared database after migration.

# Impact
Complex maintenance activities and increased interdependence among teams responsible for specific microservices.

# Linked Risks
- R65

# Stakeholders

- Engineers responsible for migration; teams in charge of specific microservices

# Contextual Constraints

- Applies when a central shared database is kept in the microservices architecture.


# Detection Source
Database Diagrams / Architecture Diagram

# Standardized Risk Entry
If a central shared database is kept after microservice migration, data coupling may make maintenance complex and increase interdependence among microservice teams.

# Key Quote for Prompting
> "A central shared database also can be kept, however, this lead to data coupling among microservices, making complex activities of maintenance."

# Logic Pattern
IF a shared database is kept AND multiple microservices depend on it THEN data coupling may increase maintenance complexity and team interdependence.
