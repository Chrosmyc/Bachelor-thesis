# Description
Features that were previously scattered over multiple subsystems may lead to product gaps or difficult governance if not gathered into coherent logical units in the target architecture.

# Root Cause
Business features are distributed across multiple legacy subsystems and have functional dependencies.

# Trigger Event
Mapping or designing target components without consolidating related features.

# Impact
Product gaps, difficult architecture governance, duplicated or inconsistent functionality in the modernized system.

# Linked Risks
- R92; R95

# Stakeholders

- Architecture Board; Business Change Management team; software architects; application developers

# Contextual Constraints

- Payments-domain SOA migration with scattered features such as interest calculation and product agreements.


# Detection Source
Architecture Diagram; Documentation; Workshops

# Standardized Risk Entry
During SOA modernization, scattered legacy functionality can create product gaps and governance problems if related features are not consolidated into logical target components.

# Key Quote for Prompting
> "related features are gathered within one logical unit to ensure that architecture governance is easy, and to minimize the product gaps"

# Logic Pattern
IF related features are scattered across subsystems AND target components do not consolidate them THEN product gaps and governance difficulties may occur.
