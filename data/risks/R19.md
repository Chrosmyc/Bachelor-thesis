# Description
Legacy systems may become dependent on proprietary cloud provider technologies and APIs.

# Root Cause
Use of proprietary cloud services and APIs

# Trigger Event
Tight integration with a specific cloud provider

# Impact
Reduced portability, higher future migration costs, interoperability limitations

# Linked Risks
- Interoperability issues

# Stakeholders

- Software Architect
- Project Manager

# Contextual Constraints

- Mainly occurs in provider-specific implementations


# Detection Source
Architecture Diagram paper

# Standardized Risk Entry
Proprietary cloud services can create vendor lock-in and reduce system portability.

# Key Quote for Prompting
> “Reducing vendor lock-in”

# Logic Pattern
IF proprietary cloud APIs are tightly coupled THEN vendor lock-in risk occurs
