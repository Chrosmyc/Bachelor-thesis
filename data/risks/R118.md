# Description
When a legacy system is replaced with a commercial off-the-shelf package, the new package may not implement all functions that the legacy system performs. This is problematic because mission-relevant functionality may be lost and the replacement may fail to achieve the expected outcome.

# Root Cause
Decades of legacy-system modification created customized functionality that cannot simply be swapped out.

# Trigger Event
Selecting or implementing a COTS replacement for a mission-critical legacy system.

# Impact
Functional gaps, reduced mission fit, rework, project failure, or failure to achieve the desired modernization outcome.

# Linked Risks
- R119; R120

# Stakeholders

- Agency or department mission owners; end users; modernization team; COTS vendor; program leadership

# Contextual Constraints

- Applies when the legacy system has been modified over many years and its full functional scope is not fully known.


# Detection Source
Application Estate Analysis; Interviews; Source Code; Project Plan

# Standardized Risk Entry
In a COTS replacement, customized legacy functionality may not be covered by the target package, creating functional gaps and threatening the modernization outcome.

# Key Quote for Prompting
> Paraphrased evidence: COTS may not do everything the legacy system does.

# Logic Pattern
IF a COTS replacement is selected AND the legacy system contains customized or poorly documented functionality THEN functional coverage gaps may threaten modernization success.
