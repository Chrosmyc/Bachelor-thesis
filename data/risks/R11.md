# Description
The project succeeds in re-hosting but fails to address the underlying legacy language (COBOL), leaving technical debt in place

# Root Cause
High complexity of code conversion; previous failed attempts at automated conversion

# Trigger Event
Completion of infrastructure migration without subsequent code modernization

# Impact
Continued reliance on legacy skills; only "partial" achievement of modernization goals.

# Linked Risks

# Stakeholders

- IT Director
- Software Architect

# Contextual Constraints

- Original code is high in complexity


# Detection Source
Source Code / Project Plan

# Standardized Risk Entry
Modernization projects often only achieve re-hosting, failing to phase out legacy codebases like COBOL due to conversion complexity

# Key Quote for Prompting
> "In two of the cases... the expected benefit (i.e., 'Phase out legacy technology') was only partially met"

IF [High Code Complexity] AND [Resource Constraints] THEN [Partial Goal Achievement]
