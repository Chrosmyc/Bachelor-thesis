# Description
AI-generated outputs for test generation, code acceleration, unit testing, or synthetic data may propagate through engineering pipelines without adequate validation.

# Root Cause
Lack of validation and controlled governance for GenAI-assisted engineering outputs.

# Trigger Event
AI-generated outputs are integrated into engineering pipelines without mandatory human review or acceptance criteria.

# Impact
Defective AI-generated outputs may enter modernization artifacts or production systems.

# Linked Risks
- R112
- R113

# Stakeholders

- Developers
- test engineers
- reviewers
- security governance
- engineering leads

# Contextual Constraints

- Applies when GenAI tools are used for code
- tests
- synthetic data
- or automation in modernization.


# Detection Source
Source Code / Project Plan

# Standardized Risk Entry
AI-assisted modernization introduces risk when generated outputs move through engineering pipelines without validation. Mandatory review and evidence are needed before integration.

# Key Quote for Prompting
> "The rapid adoption of generative AI tools ... has introduced a new governance risk: AI-generated outputs propagating through engineering pipelines without adequate validation or controlled update processes."

# Logic Pattern
IF GenAI output is produced AND it bypasses validation checkpoints THEN defective AI-generated output may propagate through modernization pipelines.
