# Description
Outsourcing an entire feature can be risky if it does not contribute to strategic objectives, does not follow outsourcing guidelines, or is not done via strategic partners.

# Root Cause
Outsourcing is only viable under strict strategic and partner conditions.

# Trigger Event
Outsourcing a feature when another realization option is viable or when outsourcing criteria are not fulfilled.

# Impact
Misalignment with strategic objectives, reduced control, inappropriate allocation of migration work.

# Linked Risks
- R90; R92

# Stakeholders

- Lead architects; strategic partners; program management committee; business owners

# Contextual Constraints

- Low-business-value features that can be developed cheaply
- such as card authorization and card payments.


# Detection Source
Project Plan; Interview

# Standardized Risk Entry
Outsourcing modernization work can create strategic and control risks unless it is limited to suitable low-priority features and strategic partners under defined guidelines.

# Key Quote for Prompting
> "only when no other option is viable"

# Logic Pattern
IF a feature is outsourced without meeting strategic outsourcing criteria THEN outsourcing-related migration risk may occur.
