# Description
If legacy functionality remains active while equivalent microservices are developed, functionality may coexist in two locations and create maintenance issues.

# Root Cause
Previous legacy iteration is not terminated as microservice development progresses.

# Trigger Event
Equivalent functionality exists in both the legacy system and newly developed microservices.

# Impact
Maintenance issues from duplicated functionality locations.

# Linked Risks
- R86

# Stakeholders

- Development teams
- software architects
- maintainers

# Contextual Constraints

- Applies during incremental development and transition to microservices.


# Detection Source
Source Code / Version Control / Project Plan

# Standardized Risk Entry
During incremental migration, keeping equivalent functionality in both legacy and microservice locations can create maintenance problems.

# Key Quote for Prompting
> "the previous iteration of the legacy system must be terminated ... to prevent the emergence of maintenance issues"

# Logic Pattern
IF microservice functionality is created AND old equivalent functionality remains active THEN duplicated-maintenance risk arises.
