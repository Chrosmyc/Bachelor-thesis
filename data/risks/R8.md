# Description
The risk of transferring problems or anti-patterns from the legacy system into the new architecture during modernization.

# Root Cause
Poor understanding of legacy logic and lack of verification.

# Trigger Event
Decomposition of legacy monolith into services

# Impact
Modern system inherits technical debt, poor performance, or maintainability issues.

# Linked Risks
- Performance Degradation.

# Stakeholders

- Software Architect
- DevOps Engineer

# Contextual Constraints

- Systems with high coupling


# Detection Source
Source Code / Architecture Diagram

# Standardized Risk Entry
Incomplete decomposition of legacy systems often leads to the preservation of existing anti-patterns in the new modernized architecture

# Key Quote for Prompting
> "...correctness of modern systems... refers to avoiding transferring problems from the legacy system during modernization, such as antipatterns."

# Logic Pattern
IF [Highly Coupled Legacy] AND [Direct Decomposition] THEN [Transfer of Legacy Flaws]
