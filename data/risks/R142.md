# Description
Modernization may be delayed or inconsistently executed when agencies or teams use different personal approaches instead of a common modernization process.

# Root Cause
Lack of a well-defined modernization process and unified approach.

# Trigger Event
Agencies plan or execute modernization without a shared guide, artifacts, or common terminology.

# Impact
Delays, inconsistent planning, harder integration of modernized applications, reduced confidence, and increased migration risk.

# Stakeholders

- Agencies
- DIR
- state CIOs
- business and IT teams

# Contextual Constraints

- Applies to cross-agency or agency-wide modernization programs requiring coordination.


# Detection Source
Project Plan

# Standardized Risk Entry
If a modernization program lacks a structured common process and shared artifacts, migration may become inconsistent, delayed, and harder to integrate across agency boundaries.

# Key Quote for Prompting
> "A well-defined modernization process can reduce risk."

# Logic Pattern
IF modernization teams use fragmented approaches AND migration/integration work must be coordinated THEN migration risk and delay risk increase.
