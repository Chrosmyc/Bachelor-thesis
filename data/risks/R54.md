# Description
In servicing, only tactical changes such as patches, code changes, and wrappers are possible; these may further deteriorate the architecture.

# Root Cause
Architecture coherence has already degraded and available knowledge is limited.

# Trigger Event
A tactical patch or wrapper is applied.

# Impact
Architecture deteriorates further and future changes become more difficult.

# Detection Source
Source Code / Architecture Diagram / Change Log

# Standardized Risk Entry
Tactical servicing changes can address immediate needs but risk further architectural decay.

# Key Quote for Prompting
> During the servicing stage only minor tactical program changes can be done. They usually take a form of patches and wrappers, and they further deteriorate the architecture.

# Logic Pattern
IF a system is in servicing AND changes are implemented as patches/wrappers THEN architecture deterioration may increase.
