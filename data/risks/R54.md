# Description
Major evolution changes may be required that were not predicted during initial design; implementing them can compromise system integrity and invariants.

# Root Cause
Future requirements and environments cannot be fully predicted.

# Trigger Event
A major new requirement or environmental change must be implemented.

# Impact
Architecture integrity may be damaged and future evolution becomes harder.

# Contextual Constraints



# Detection Source
Architecture Diagram / Source Code / Project Plan

# Standardized Risk Entry
When major unanticipated changes are implemented without architecture support, modernization can damage architectural integrity.

# Key Quote for Prompting
> The aim is to implement (and revalidate) possibly major changes to the system without being able a priori to predict how user requirements will evolve.

# Logic Pattern
IF architecture does not support unanticipated change AND a major requirement is implemented THEN integrity may be compromised.
