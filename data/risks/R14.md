# Description
The new graphical interface is merely a wrapper around old text screens. The underlying system remains as inflexible and difficult to maintain as before.

# Root Cause
Modernization only addresses the surface (UI) without touching the legacy core or logic.

# Trigger Event
Changes in the legacy system's text output or the need for functional backend changes.

# Impact
Long-term maintenance costs remain high; technical debt is preserved; "whipped cream on road kill" effect.

# Linked Risks
- Cost Overrun

# Stakeholders

- Software Architect
- Project Manager
- IT Department

# Contextual Constraints

- Applies when screen scraping is used as the sole modernization technique.


# Detection Source
Architecture Diagram / Project Plan

# Standardized Risk Entry
Screen scraping improves usability but fails to reduce backend complexity, leading to continued high maintenance efforts and system fragility.

# Key Quote for Prompting
> "From the IT department's perspective, the new system is as inflexible and difficult to maintain as the legacy system."

# Logic Pattern
IF [Modernization = Screen Scraping] AND [Objective = Long-term Maintainability] THEN [Risk = Preservation of Inflexibility]
