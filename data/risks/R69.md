# Description
SOA refactoring may remain incomplete if a clean access layer to the core/internal layer is not sufficiently structured.

# Root Cause
Core functionality still needs clean access-layer structuring.

# Trigger Event
Refactoring an existing application to SOA.

# Impact
Maintainability or integration quality may be limited.

# Linked Risks

# Stakeholders

- Architects; developers

# Contextual Constraints

- Layered SOA migration around an existing core.


# Detection Source
Literature Review

# Standardized Risk Entry
SOA refactoring may remain incomplete if a clean access layer to the core/internal layer is not sufficiently structured.

# Key Quote for Prompting
> "more is needed to add to the core functionalities that required the structuring of a clean access layer to the core or internal layer."

# Logic Pattern
IF SOA refactoring lacks a clean core access layer THEN maintainability and integration risk occurs.
