# Description
SOA increases the complexity and number of services, potentially leading to a "spaghetti dish" of services if not managed.

# Root Cause
Poorly managed relationships between services and sub-services; lack of service orchestration.

# Trigger Event
Growing the number of services without a strong governance framework.

# Impact
Loss of agility; difficulty in tracing and tracking; management overhead.

# Linked Risks
- Governance Failure

# Stakeholders

- Software Architect
- DevOps Engineer

# Contextual Constraints



# Detection Source
Architecture Diagram / Monitoring Tools

# Standardized Risk Entry
Without central orchestration and governance, the high number of new services leads to unmanageable complexity and lost agility.

# Key Quote for Prompting
> "SOA will increase the complexity and the number of services required other than governance and coordination."

# Logic Pattern
IF [Number of services grows] AND [Governance is weak] THEN [Complexity increases / Agility is lost]
