# Description
Wrapped web services may retain the state of the last caller, so data from different processes can be mixed.

# Root Cause
The wrapped service state is not safely separated per caller.

# Trigger Event
Multiple processes use the same wrapped service.

# Impact
Incorrect service behavior or mixed data state.

# Linked Risks

# Stakeholders

- Developers; architects; operations

# Contextual Constraints

- Shared wrapped service usage.


# Detection Source
Literature Review

# Standardized Risk Entry
Wrapped web services may retain the state of the last caller, so data from different processes can be mixed.

# Key Quote for Prompting
> "The state of the data contained within a wrapped web service is that of the last caller. Thus, if different processes are using the same service, their data will be mixed."

# Logic Pattern
IF a wrapped service stores caller state AND multiple processes use it THEN data mixing risk occurs.
