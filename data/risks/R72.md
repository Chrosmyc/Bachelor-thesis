# Description
A single LLM prompt can produce plausible modern code but fails to preserve behavioral equivalence because understanding, specification, transformation, and validation are collapsed into one step.

# Root Cause
Single-pass prompting lacks intermediate representation, verification, and feedback mechanisms.

# Trigger Event
A team asks an LLM to convert a legacy artifact directly into a modern API or service.

# Impact
Generated code may compile but fail behavioral tests and preserve no gold-standard business rules.

# Linked Risks
- R71

# Stakeholders

- Developers
- modernization teams
- QA teams
- business owners

# Contextual Constraints

- Applies to single-prompt or monolithic LLM modernization workflows.


# Detection Source
Generated code, gold-standard tests, behavioral equivalence metrics

# Standardized Risk Entry
If modernization relies on a single LLM prompt without specification and validation, the result may look plausible but fail behavioral equivalence tests.

# Key Quote for Prompting
> "SP-LLM and CoT-LLM scored 0.0% on every scenario, on every backbone."

# Logic Pattern
IF LLM modernization is single-pass AND no BSG or validation feedback is used THEN behavioral preservation failure risk is high
