# Description
Migration complexity increases when data categories, dependencies, and mappings are not properly classified before execution.

# Root Cause
Critical, regulatory, customer transaction, historical, and analytical data have different migration requirements.

# Trigger Event
Migration starts before data classification and dependency mapping are completed.

# Impact
Migration complexity, timeline deviation, incorrect mappings, and governance issues.

# Linked Risks
- R58; R60

# Stakeholders

- Data architects; governance teams; migration teams

# Contextual Constraints

- Large financial data estates.


# Detection Source
Project Plan / Operational Data

# Standardized Risk Entry
Poor data classification and mapping is a modernization risk because it increases migration complexity and weakens timeline adherence.

# Key Quote for Prompting
> proper data classification reduces migration complexities by approximately 65%

# Logic Pattern
IF data dependencies and categories are not classified AND migration begins THEN mapping and complexity risk increases
