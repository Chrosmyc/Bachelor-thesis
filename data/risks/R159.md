# Description
Automated AI tools output inaccurate code fragments, leading to operational compilation or runtime errors in the modern system.

# Root Cause
Complex, verbose, and undocumented nature of legacy languages like COBOL translated via LLMs without verification.

# Trigger Event
Direct injection of AI-generated code or architecture changes into production streams.

# Impact
Introduction of new technical bugs, system inaccuracy, unexpected business logic deviation.

# Linked Risks

# Stakeholders

- Developers
- QA Engineers

# Contextual Constraints

- Valid when using generative AI or automated translation tools for language conversion (e.g.
- COBOL to Java).


# Detection Source
Source Code

# Standardized Risk Entry
Relying blindly on automated AI code conversion introduces subtle semantic logic errors and software bugs into the target platform.

# Key Quote for Prompting
> "As with any Al application, teams must still exercise caution, verifying the outputs for accuracy and testing any generated code..."

# Logic Pattern
IF [Generative AI translates code] AND [Human verification is skipped] THEN [Semantic Regressions via AI Translation Error]
