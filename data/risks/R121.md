# Description
External partners may spend considerably more time learning the application portfolio and ways of working, increasing cost, risk, and project length.

# Root Cause
Insufficient partner understanding of portfolio, domain knowledge, and working practices

# Trigger Event
Partners begin modernization without enough internal context transfer

# Impact
Increased cost, higher risk, and longer modernization project duration

# Linked Risks
- R120

# Stakeholders

- External partners
- internal project owners
- IT decision makers
- application teams

# Contextual Constraints

- Applies when partner teams are not guided by knowledgeable internal teams


# Detection Source
Project plan; onboarding duration; knowledge-transfer status

# Standardized Risk Entry
Partner-led modernization can become slower and riskier when partners must spend additional time learning the portfolio and ways of working.

# Key Quote for Prompting
> "increasing the cost, risk, and length of the project"

# Logic Pattern
IF external partners lack portfolio context AND knowledge transfer is insufficient THEN cost, risk, and project length may increase.
