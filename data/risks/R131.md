# Description
Security and identity settings must be replicated or adjusted in Azure; if they are not inventoried, migrated workloads may suffer authentication, encryption, firewall, or access-control issues.

# Root Cause
Missing inventory of service accounts, user accounts, API keys, hard-coded credentials, encryption methods, firewall rules, and access control lists.

# Trigger Event
Workloads are migrated without complete security and identity configuration documentation.

# Impact
Authentication problems, insecure access policies, encryption misalignment, firewall misconfiguration, or migration sequencing issues.

# Linked Risks

# Stakeholders

- Security team
- identity team
- migration team
- workload owners

# Contextual Constraints

- Applies when security and identity configurations must be replicated or adjusted in Azure.


# Detection Source
Architecture Diagram / Interview / Operational Data

# Standardized Risk Entry
Missing security and identity inventories can cause authentication, encryption, firewall, or access-control failures when workloads are moved to Azure.

# Key Quote for Prompting
> "These configurations need to be replicated or adjusted in Azure"

# Logic Pattern
IF security and identity configurations are not inventoried AND workloads migrate to Azure THEN access or security controls may fail.
