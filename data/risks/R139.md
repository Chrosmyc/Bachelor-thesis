# Description
Overly complex components selected for replacement can prolong migration, increase errors, and discourage the team and stakeholders.

# Root Cause
Slice selection may choose components beyond the team’s current capacity, dependency knowledge, or delivery confidence.

# Trigger Event
The team selects a highly complex component for an early or poorly prepared migration slice.

# Impact
Longer migration time, more errors, lower morale, stakeholder discouragement, and risk of failed delivery.

# Linked Risks
- R136; R145; R154

# Stakeholders

- Development team; stakeholders; project managers; users

# Contextual Constraints

- Thin-slice migration where component complexity is high.


# Detection Source
Architecture Diagram / Project Plan / Backlog

# Standardized Risk Entry
If a modernization slice is overly complex, it can prolong the migration, increase errors, and weaken team and stakeholder confidence.
