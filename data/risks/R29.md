# Description
Data shared between legacy and modernized systems may become inconsistent during transaction-based migration.

# Root Cause
Shared tables are modified by both systems.

# Trigger Event
Parallel operation of legacy and modernized transactions.

# Impact
Data inconsistency, operational instability.

# Linked Risks
- Operational Failure

# Stakeholders

- Database Architect
- DevOps Engineer

# Contextual Constraints

- Applies when both systems access the same data simultaneously.


# Detection Source
Database Schema

# Standardized Risk Entry
Shared access to synchronized legacy and modernized databases can create consistency and synchronization risks.

# Key Quote for Prompting
> "Changes that occur in one location must be propagated to the other."

# Logic Pattern
IF legacy and modernized systems update shared data simultaneously THEN synchronization risks emerge.
