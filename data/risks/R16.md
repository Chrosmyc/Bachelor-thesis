# Description
Failure to plan for how the legacy system will be decommissioned once the new system is active

# Root Cause
Focus only on building the new system without considering the data migration and shutdown of the old one.

# Trigger Event
Completion of new system development without a transition plan.

# Impact
Continued O&M costs for the old system (dual-running costs), data inconsistency.

# Linked Risks
- Increased attack surface (maintaining two systems)
- Cost overrun

# Stakeholders

- Software Architect
- DevOps Engineer
- Financial Officer

# Contextual Constraints



# Detection Source
Modernization Roadmap

# Standardized Risk Entry
Absence of a clear disposition plan results in the inability to retire legacy systems, leading to redundant operational costs

# Key Quote for Prompting
> "Documentation of modernization plans should include... a plan for the disposition of the legacy system."

# Logic Pattern
IF [No disposition strategy exists] THEN [Legacy system remains active indefinitely alongside new system].
