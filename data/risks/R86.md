# Description
The modernization may produce microservices that are too large, too small, too coupled, or insufficiently cohesive.

# Root Cause
Difficulty determining the optimal size and boundaries of microservices.

# Trigger Event
The legacy system is divided into services without clear low coupling, high cohesion, and bounded context criteria.

# Impact
The modernized architecture may not meet business capabilities effectively and may increase complexity.

# Linked Risks
- R85
- R88

# Stakeholders

- Software architects
- development teams
- domain specialists

# Contextual Constraints

- Applies during decomposition and microservice candidate selection.


# Detection Source
Architecture Diagram / Source Code / Project Plan

# Standardized Risk Entry
Incorrect microservice granularity can undermine the modernized architecture by creating services that lack clear scope, cohesion, and business capability alignment.

# Key Quote for Prompting
> "the primary challenge associated with this activity is determining the optimal size for microservices"

# Logic Pattern
IF microservices are not designed with low coupling, high cohesion, and bounded context THEN wrong granularity risk materializes.
