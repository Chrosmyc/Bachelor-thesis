# Description
Difficulty in translating monolithic/procedural legacy semantics into modern, hierarchic, and structured object-oriented or component semantics.

# Root Cause
Fundamental architectural differences between procedural legacy code and modern OO patterns.

# Trigger Event
Mapping legacy functions to objects or components.

# Impact
Incorrect business logic representation, increased complexity, potential system failure during execution.

# Linked Risks
- Functional Errors
- Project Delay.

# Stakeholders

- Software Architect
- Lead Developer

# Contextual Constraints

- Higher risk when developers lack deep domain knowledge of the legacy system.


# Detection Source
Source Code / Architecture Diagram

# Standardized Risk Entry
The shift from procedural to object-oriented logic during modernization often fails due to the difficulty of mapping monolithic semantics to modern structures.

# Key Quote for Prompting
> "Translating the monolithic and plain semantics of the often procedural legacy system to the richly hierarchic and structured semantics of an object-oriented system can be a difficult task."

# Logic Pattern
IF [Strategy = OO-Wrapping] AND [Source = Procedural Legacy] THEN [Risk = Semantic Mapping Failure]
