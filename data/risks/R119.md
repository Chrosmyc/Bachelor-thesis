# Description
Replacement and rewrite efforts can be undermined when the organization does not fully know what the legacy system does or how it does it. The risk is not the legacy system itself, but that hidden behavior becomes an unknown requirement during modernization.

# Root Cause
Incomplete understanding of legacy application behavior, dependencies, and implementation details.

# Trigger Event
Starting replacement or rewrite activities before the application estate has been sufficiently analyzed.

# Impact
Missing requirements, incorrect target design, functional regression, rework, cost escalation, schedule delay, or failed modernization.

# Linked Risks
- R118; R120; R121; R123; R126

# Stakeholders

- Modernization team; architects; developers; product owners; end users; mission owners

# Contextual Constraints

- Applies especially to large
- complex
- long-lived codebases and portfolios.


# Detection Source
Application Estate Analysis; Source Code; Architecture Diagram; Interviews; Project Plan

# Standardized Risk Entry
Hidden or poorly understood legacy behavior can become an unknown requirement in replacement or rewrite projects, increasing the risk of incorrect design, rework, and modernization failure.

# Key Quote for Prompting
> Paraphrased evidence: modernization suffers when organizations do not know what they do not know.

# Logic Pattern
IF legacy behavior is insufficiently understood AND replacement or rewrite begins THEN unknown functionality may become missing requirements and create modernization failure risk.
