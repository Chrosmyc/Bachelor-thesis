# Description
Operating systems, middleware, and application frameworks that are unsupported, deprecated, or near end of support can block migration or cause post-migration runtime problems.

# Root Cause
Components do not align with Azure support policies.

# Trigger Event
The workload is migrated before compatibility issues are detected and remediated.

# Impact
Migration blockers, runtime failures, significant rework, or delayed migration.

# Linked Risks

# Stakeholders

- Migration team
- application team
- operations team
- vendors

# Contextual Constraints

- Applies to OS
- middleware
- and application framework compatibility checks against Azure support policies.


# Detection Source
Source Code / Operational Data / Project Plan

# Standardized Risk Entry
Unsupported or deprecated workload components can become Azure migration blockers or cause runtime failures if not detected before migration.

# Key Quote for Prompting
> "unsupported, deprecated, or approaching end of support"

# Logic Pattern
IF unsupported components exist AND migration proceeds without compatibility assessment THEN migration blockers or runtime failures may occur.
