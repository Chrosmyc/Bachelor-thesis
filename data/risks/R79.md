# Description
The SetTaskAlert service requires event handling in a service-oriented environment, but the paper states this had only recently been introduced in SOA 2.0.

# Root Cause
Required SOA event-handling capability is immature or newly introduced.

# Trigger Event
Choosing to implement an alert/event-based service as part of the pilot migration.

# Impact
Unjustifiable technical risk for the pilot, possible implementation failure, and schedule pressure.

# Linked Risks
- R80

# Stakeholders

- Service provider
- service consumer
- developers
- architects

# Contextual Constraints

- Specific to event-based services such as SetTaskAlert in the MSS migration case.


# Detection Source
Project Plan / Interview / Architecture Diagram

# Standardized Risk Entry
Event-based service migration may be risky when the required SOA event-handling technology is immature.

# Key Quote for Prompting
> "The SetTaskAlert service requires the handling of events in service-oriented environments which is something that has been recently introduced in SOA 2.0"

# Logic Pattern
IF a candidate service requires immature SOA event handling AND it is included in a short pilot THEN implementation risk increases.
