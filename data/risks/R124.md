# Description
A workload is rehosted even though modernization is likely within the next two years. The initial rehost can become temporary work that must be redone through refactoring or rearchitecting later.

# Root Cause
Near-term modernization need is not assessed before choosing rehost.

# Trigger Event
The organization rehosts a workload that will soon need modernization.

# Impact
Duplicate effort, rework, additional cost, and delayed realization of modernization benefits.

# Linked Risks
- R124

# Stakeholders

- Business owners
- technical leads

# Contextual Constraints

- Applies when modernization is likely in the near future.


# Detection Source
Project Plan / Interview / workload roadmap

# Standardized Risk Entry
Rehosting a workload that will require modernization soon can create duplicate effort and unnecessary rework.

# Key Quote for Prompting
> [Paraphrased evidence from G10] Rehosting should be used only when the workload can remain unchanged for at least two years.

# Logic Pattern
IF modernization is likely within two years AND the workload is rehosted THEN duplicate modernization effort may occur.
