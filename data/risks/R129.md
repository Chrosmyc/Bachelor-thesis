# Description
Business-critical applications may face disruption if rapid cloud uptake is attempted without keeping changes minimal.

# Root Cause
The guidance positions rehost/replatform for rapid cloud uptake with minimal changes to business-critical applications.

# Trigger Event
Moving business-critical applications to cloud while introducing more changes than necessary.

# Impact
Disruption of business-critical application modernization.

# Linked Risks

# Stakeholders

- Application owners
- operations teams
- business stakeholders

# Contextual Constraints

- Applies to business-critical legacy applications requiring rapid cloud uptake.


# Detection Source


# Standardized Risk Entry
Rapid cloud uptake can create disruption risk for business-critical applications if modernization introduces more changes than the rehost/replatform approach requires.

# Key Quote for Prompting
> Source cue: rapid cloud uptake with minimal changes for business-critical applications.

# Logic Pattern
IF rapid cloud uptake is required AND modernization introduces non-minimal changes THEN change-disruption risk may occur.
