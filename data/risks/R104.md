# Description
Replacing with a COTS product frees agencies from custom development but creates dependency on software upgrades and exposure to license fee increases.

# Root Cause
Vendor-controlled product roadmap, software upgrades, and licensing model.

# Trigger Event
Agency replaces legacy application with COTS solution.

# Impact
Vendor dependency, update constraints, license fee increases, process adaptation needs.

# Linked Risks

# Stakeholders

- Agency; vendor; procurement; finance; business owners

# Contextual Constraints

- COTS replacement strategy.


# Detection Source
Project Plan

# Standardized Risk Entry
COTS replacement reduces custom development but may introduce dependency on vendor upgrades and future license costs.

# Key Quote for Prompting
> "dependent on software upgrades and subject to license fee increases"

# Logic Pattern
IF modernization replaces custom legacy functionality with COTS THEN vendor upgrade and license dependency risk may arise.
