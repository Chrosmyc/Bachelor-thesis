# Description
Unpredicted requests for change during migration can disrupt the migration process if they are not systematically resolved and escalated according to business priority and cost.

# Root Cause
The migration process contains many dependencies and changing business/architecture priorities.

# Trigger Event
A Request For Change appears during migration.

# Impact
Misaligned decisions, delayed migration, unmanaged cost escalation, architectural inconsistency.

# Linked Risks
- R90; R92

# Stakeholders

- Business Change Management team; Architecture Board; Core Team; Steering Committee

# Contextual Constraints

- Large-scale migration with business-priority thresholds and architectural governance needs.


# Detection Source
Project Plan; Workshops; Interview

# Standardized Risk Entry
During large-scale modernization, unpredicted change requests can disrupt cost, schedule, and architecture alignment unless they are resolved through defined governance paths.

# Key Quote for Prompting
> "any unpredicted changes were systematically resolved"

# Logic Pattern
IF an RFC arises during migration AND there is no systematic governance process THEN uncontrolled migration change risk increases.
