# Description
In gradual replacement, poor architecture planning can result in too many integrations and components.

# Root Cause
Architecture is not planned well during component-by-component modernization.

# Trigger Event
Replacing components with new technologies without a well-planned architecture.

# Impact
Too many integrations and components.

# Linked Risks
- R151

# Stakeholders

- Enterprise architects; modernization team; application development companies; operations

# Contextual Constraints

- Applies when gradual replacement introduces multiple technologies
- integrations
- or components.


# Detection Source
Architecture diagram / project plan

# Standardized Risk Entry
Poor architecture planning during gradual replacement can create excessive integration and component complexity.

# Key Quote for Prompting
> "if the architecture is not planned well, it may result in too many integrations and components."

# Logic Pattern
IF gradual replacement is used AND architecture is not planned well THEN too many integrations and components may result.
