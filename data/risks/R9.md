# Description
Users who are accustomed to highly efficient legacy character-based interfaces may find modern GUIs slower or less intuitive.

# Root Cause
Users are optimized for specific workflows and do not want to change

# Trigger Event
Deployment of the new graphical user interface

# Impact
Lower user satisfaction, initial drop in productivity, and complaints during transition.

# Linked Risks

# Stakeholders

- Project Manager
- End Users

# Contextual Constraints

- Specifically high when moving from terminal-based (console) to Windows-based GUI.


# Detection Source
Interview Transcripts

# Standardized Risk Entry
Modernizing character-based interfaces to GUI often triggers significant user resistance due to established workflow habits.

# Key Quote for Prompting
> "Most users were not really happy when going to the Windows... surroundings. Because they were so used to those [character-based] screens."

# Logic Pattern
IF [Modernizing UI] AND [Users used to old system] THEN [User Resistance]
