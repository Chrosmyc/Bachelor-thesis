# Description
Assessment tools can automate discovery, but if migration planning relies only on tool output, undocumented dependencies may remain invisible and later affect migration sequencing or integration.

# Root Cause
Automated tools might miss undocumented dependencies.

# Trigger Event
The migration team uses automated discovery without workload-owner validation or interviews.

# Impact
Missing dependencies can lead to inaccurate planning, broken integrations, poor migration wave grouping, or service disruption.

# Linked Risks
- R133

# Stakeholders

- Migration team
- workload owners
- application owners
- architects

# Contextual Constraints

- Applies when automated assessment data is used without manual validation.


# Detection Source
Architecture Diagram / Interview / Operational Data

# Standardized Risk Entry
If automated workload discovery is used without expert validation, undocumented dependencies may remain undetected and undermine migration planning or integration reliability.

# Key Quote for Prompting
> "they might miss undocumented dependencies"

# Logic Pattern
IF automated discovery is used alone AND undocumented dependencies exist THEN migration planning may miss critical dependencies.
