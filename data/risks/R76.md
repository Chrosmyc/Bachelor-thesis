# Description
Legacy applications designed for single-tenant environments may experience security, performance, availability, or customization issues after migration to multi-tenant cloud environments.

# Root Cause
Legacy systems are often designed using a single-tenant architecture while cloud environments commonly rely on multi-tenancy.

# Trigger Event
Migration of a single-tenant legacy application to a multi-tenant cloud environment.

# Impact
Performance degradation, security concerns, availability issues, and reduced customizability.

# Linked Risks

# Stakeholders

- Application developers; Cloud architects; End users

# Contextual Constraints

- Applies when legacy applications are migrated to cloud environments utilizing shared resources among multiple tenants.


# Detection Source
Architecture Diagram

# Standardized Risk Entry
Legacy applications designed for single-tenant environments may experience non-functional issues when migrated to multi-tenant cloud architectures.

# Key Quote for Prompting
> "Many legacy systems are built with a single-tenant architecture which can result in several problems related to performance, security, availability, and customizability when migrated to cloud."

# Logic Pattern
IF single-tenant application is migrated to a multi-tenant cloud environment THEN multi-tenancy-related risks increase
