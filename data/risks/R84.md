# Description
During implementation, teams may face problems with common code shared across microservices and API changes that break consumers.

# Root Cause
Microservice implementation requires code management across independently deployed services.

# Trigger Event
Developers modify shared code or service APIs without mechanisms for managing compatibility and consumers.

# Impact
Broken service interactions; integration failures; rework; delayed migration.

# Linked Risks
- R83

# Stakeholders

- Developers; engineers; architects

# Contextual Constraints

- Applies during execution and integration of microservices.


# Detection Source
Source Code / API Specification / Project Plan

# Standardized Risk Entry
If shared code and API evolution are not managed, modernization may introduce breaking changes between microservices.

# Key Quote for Prompting
> Challenges like managing common code shared by multiple microservices and preventing/dealing with breaking API changes will be faced by developers.

# Logic Pattern
IF shared code or APIs change AND compatibility management is missing THEN risk of breaking API changes occurs.
