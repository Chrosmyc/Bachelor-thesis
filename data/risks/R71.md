# Description
Modernized code may compile and run while silently changing the legacy system's behavior, especially when modernization is treated as syntax translation rather than behavioral preservation.

# Root Cause
Business behavior is embedded in control flow, conditions, exception handlers, and configuration rather than explicit documentation.

# Trigger Event
A legacy artifact is translated directly into a modern implementation without an explicit behavioral specification.

# Impact
Silent production failures, altered validation logic, broken business rules, and a modern system that behaves differently from the original.

- Modernization engineers

# Contextual Constraints

- Applies when modernization focuses on code translation rather than preservation of behavioral contracts.


# Detection Source
Generated code, behavioral tests, equivalence reports

# Standardized Risk Entry
If a modernization process translates legacy code without explicit behavioral preservation, the modernized system may compile but silently diverge from required business behavior.

# Key Quote for Prompting
> "A line-by-line COBOL-to-Java translation can compile and run yet silently break edge-case handling, alter validation logic, or drop constraints that were never written down."

# Logic Pattern
IF modernization is direct syntax translation AND behavioral contracts are not made explicit THEN behavioral semantics may be lost
