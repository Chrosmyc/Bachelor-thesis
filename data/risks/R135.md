# Description
The incremental Strangler Fig approach may extend the transition period, requiring old and new systems to coexist longer.

# Root Cause
Incremental replacement intentionally keeps both systems active during migration.

# Trigger Event
Migration proceeds over an extended period before full replacement and decommissioning.

# Impact
Prolonged costs and greater complexity in managing two systems simultaneously.

# Linked Risks
- R140; R141; R150

# Stakeholders

- Project managers; operations; developers; business owners

# Contextual Constraints

- Incremental modernization with coexisting legacy and modern components.


# Detection Source
Project Plan / Operational Data

# Standardized Risk Entry
If incremental modernization keeps old and new systems active for a long time, the project may incur prolonged costs and dual-system management complexity.

> 

# Logic Pattern
IF incremental migration extends over time AND both systems must operate concurrently THEN dual-system transition risk increases.
