# Description
Modernization scope can be misunderstood when systems exchange data with other agencies or systems that cannot accept changed formats.

# Root Cause
Current environment, interfaces, dependencies, and complexity are not fully understood.

# Trigger Event
A system is modernized while dependent systems or partners remain unchanged.

# Impact
Incorrect modernization approach, additional modernization needs, integration failures, delays.

# Linked Risks
- R100
- R101
- R102

# Stakeholders

- Program manager; system architect; data architect; partner agencies; business owners

# Contextual Constraints

- Cross-agency or cross-enterprise data exchange and integration scenarios.


# Detection Source
Architecture Diagram

# Standardized Risk Entry
Modernization may fail or expand unexpectedly when scope and dependencies across systems are not assessed before selecting an approach.

# Key Quote for Prompting
> "The first step is to understand the project scope and complexity."

# Logic Pattern
IF dependencies and data-exchange constraints are unknown AND modernization changes interfaces THEN scope and integration risks may materialize.
