# Description
The migration may proceed without sufficient understanding of source code organization, features, dependencies, database structure, data flow, technologies, documentation, business goals, and organizational aspects.

# Root Cause
Insufficient analysis of the actual legacy system and its technical, business, and organizational context.

# Trigger Event
Starting architecture definition or decomposition before the legacy system is well-understood.

# Impact
The paper does not explicitly state consequences, but insufficient understanding can weaken later identification of microservice candidates and architecture definition.

# Linked Risks
- R64; R65; R67

# Stakeholders

- Engineers responsible for migration; development teams; stakeholders

# Contextual Constraints

- Applies in the comprehension phase before defining the new architecture.


# Detection Source
Source Code / Design Models / Database Diagrams / Logs / Documentation / Static and Dynamic Information

# Standardized Risk Entry
If the legacy system is not well-understood before migration, later modernization steps may be based on incomplete knowledge of features, dependencies, data flow, and technologies.

# Key Quote for Prompting
> "The legacy system must be well-understood."

# Logic Pattern
IF legacy understanding is incomplete AND migration planning continues THEN later decomposition and architecture decisions may be poorly grounded.
