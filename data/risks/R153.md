# Description
The duct tape approach can create cost risk when multiple small changes accumulate within an application.

# Root Cause
Multiple local, small-scale changes are made to the application.

# Trigger Event
Repeatedly applying duct tape changes to address specific issues.

# Impact
Cost of multiple changes in an application.

# Linked Risks

# Stakeholders

- Management; modernization team; application owners

# Contextual Constraints

- Applies to duct tape modernization where small-scale changes accumulate over time.


# Detection Source
Project plan / budget information

# Standardized Risk Entry
The duct tape approach can produce accumulated cost risk when many small application changes are implemented.

# Key Quote for Prompting
> "The duct tape approach is associated with many risks such as the cost of multiple changes in an app."

# Logic Pattern
IF the duct tape approach is repeatedly used AND multiple application changes accumulate THEN modernization costs may increase.
