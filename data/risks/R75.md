# Description
The cloud vendor manages the underlying infrastructure and technology stack. Organizations have limited control over and visibility into the underlying technology used to host migrated applications.

# Root Cause
Cloud vendors manage the underlying technology and infrastructure.

# Trigger Event
Migration of an application or system to a cloud environment managed by a third-party provider.

# Impact
Reduced visibility into infrastructure, uncertainty regarding technology choices, potential operational risks, and limited ability to influence infrastructure-level decisions.

# Linked Risks

# Stakeholders

- Cloud provider; Application owners; IT infrastructure staff

# Contextual Constraints

- Applies when cloud services provided and managed by external providers are utilized.


# Detection Source
Architecture Diagram / Project Plan

# Standardized Risk Entry
Cloud migration can reduce organizational control and visibility over the underlying infrastructure because cloud providers manage the technology stack.

# Key Quote for Prompting
> "Cloud vendor manages underlying tech, you have no control or knowledge about it."

# Logic Pattern
IF application is migrated to a third-party cloud AND infrastructure is provider-managed THEN lack-of-control risk increases
