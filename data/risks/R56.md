# Description
Introducing modern technology without considering later change can create new maintenance problems rather than solving old ones.

# Root Cause
Maintenance implications of new technologies are not well understood before adoption.

# Trigger Event
A new technology such as OO, agents, components, GUIs, or distributed/concurrent programming is introduced.

# Impact
Maintenance can become more difficult; expected modernization benefits may not materialize.

# Detection Source
Technology Plan / Architecture Diagram

# Standardized Risk Entry
Modernization through new technology can create maintenance risks if lifecycle consequences are not assessed.

# Key Quote for Prompting
> Often new technologies are proposed and introduced without consideration of what happens when the software has to be changed.

# Logic Pattern
IF a modern technology is introduced AND maintenance implications are not evaluated THEN new maintenance problems may emerge.
