# Description
Changing requirements during modernization can disrupt the modernization process.

# Root Cause
Requirements or system decisions change during the modernization process.

# Trigger Event
People change the system or requirements while modernization is ongoing.

# Impact
Schedule pressure, additional complexity, compromised modernization quality.

# Linked Risks

# Stakeholders

- Project team
- business stakeholders

# Contextual Constraints



# Detection Source
Project Plan / Requirements

# Standardized Risk Entry
Changing requirements during modernization can disrupt the modernization process.

# Key Quote for Prompting
> "If people start to change the system or the requirements of the system while you do modernization."

# Logic Pattern
IF requirements change during modernization AND the project is already executing THEN scope and schedule risk increase.
