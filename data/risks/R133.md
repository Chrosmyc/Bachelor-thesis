# Description
Dependency data that is not centrally stored, accessible, or regularly updated can reduce cross-team collaboration and make migration planning inaccurate as the migration progresses.

# Root Cause
Dependency information is not maintained in a central repository.

# Trigger Event
Multiple teams plan migration work from inconsistent or outdated dependency information.

# Impact
Planning errors, missed dependencies, duplicated effort, weak cross-team coordination, broken dependencies.

# Linked Risks
- R130

# Stakeholders

- Migration team
- application teams
- architects
- workload owners

# Contextual Constraints

- Applies during ongoing migration planning where dependency information changes over time.


# Detection Source
Project Plan / Architecture Diagram

# Standardized Risk Entry
Dependency data that is not centralized and maintained can become stale or siloed, reducing migration planning accuracy and cross-team coordination.

# Key Quote for Prompting
> "Ensure the repository is accessible and regularly updated"

# Logic Pattern
IF dependency data is not centralized or updated AND teams plan migration waves THEN planning errors may increase.
