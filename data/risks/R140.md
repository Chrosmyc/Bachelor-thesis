# Description
Trying to replicate every legacy feature can carry inefficiencies, bugs, outdated technologies, and unnecessary complexity into the new system.

# Root Cause
Fear of losing functionality or disrupting user experience can drive uncritical replication.

# Trigger Event
The team sets exact feature parity as the default modernization goal without evaluating feature value.

# Impact
Wasted resources, reduced innovation, carryover of defects and outdated technology, and lower modernization value.

# Linked Risks
- R149; R143

# Stakeholders

- Business stakeholders; users; product owners; modernization team

# Contextual Constraints

- Modernization projects where legacy functionality is assumed to be fully replicated.


# Detection Source
Backlog / Feature Inventory

# Standardized Risk Entry
If modernization blindly replicates all legacy features, the new system may inherit inefficiencies, bugs, and outdated technology while consuming unnecessary resources.

> 

# Logic Pattern
IF feature parity is pursued blindly AND legacy features are not evaluated THEN carryover risk may materialize.
