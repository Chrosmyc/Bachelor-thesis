# Description
After workloads are migrated to AWS, the organization may not optimize them for performance and spend, risking lower value realization.

# Root Cause
Optimization is not performed after workload migration

# Trigger Event
Workloads are moved to AWS but performance and spend optimization is skipped or delayed

# Impact
Higher-than-necessary spend; suboptimal performance; delayed business value

# Linked Risks
- R114
- R115
- R117

# Stakeholders

- IT operations
- finance stakeholders
- business owners
- cloud operations teams

# Contextual Constraints

- Post-migration workload operation on AWS


# Detection Source
Operational data / cost and performance monitoring

# Standardized Risk Entry
If post-migration optimization is not performed, modernized workloads may not reach expected performance or cost-efficiency.

# Key Quote for Prompting
> "optimizing for performance and spend"

# Logic Pattern
IF workloads are migrated AND optimization is skipped THEN performance and spend outcomes may be suboptimal.
