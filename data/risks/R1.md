# Description
Errors occurring during the transfer of data from old (often non-relational) databases to modern systems, leading to data corruption or system disturbance.

# Root Cause
Use of obsolete database models that do not support modern relational structures.

# Trigger Event
Execution of the cut-over or data ETL (Extract, Transform, Load) phase.

# Impact
New system instability, loss of business-critical data, high regaining costs.

# Linked Risks
- System Downtime
- Business Disruption.

# Stakeholders

- Database Administrator
- Software Architect
- Data Analyst

# Contextual Constraints

- Old non-supported databases
- big data


# Detection Source
Database Scheme Audit

# Standardized Risk Entry
Data migration is a critical risk factor where errors in transferring data from obsolete structures to modern environments can lead to system instability and business disruption.

# Key Quote for Prompting
> "The main risk in modernization is that the data migration, which cannot be done perfectly. Errors are made and you have some risk that your new system is disturbed."

# Logic Pattern
IF [Non-Relational Database] AND [High Business Criticality] THEN [Incomplete or Erroneous Data Migration]
