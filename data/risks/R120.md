# Description
If external partners modernize applications without an internal counterpart, they may build software that cannot be used because it violates regulatory, operational, or organizational constraints.

# Root Cause
Missing internal ownership, missing domain knowledge, and insufficient communication of enterprise constraints

# Trigger Event
External partners lead or execute modernization without internal guidance

# Impact
Unusable software, wasted effort, higher risk, and failed modernization outcome

# Linked Risks
- R121

# Stakeholders

- External partners
- internal teams
- IT decision makers
- compliance/regulatory stakeholders
- operations teams

# Contextual Constraints

- Applies when partners lack internal counterpart direction and enterprise context


# Detection Source
Project governance review; partner deliverable review

# Standardized Risk Entry
Without internal ownership and constraint guidance, external partners may produce modernized software that the enterprise cannot use.

# Key Quote for Prompting
> "software that cannot be used"

# Logic Pattern
IF external partners modernize software AND no internal counterpart provides direction THEN the delivered software may be unusable.
