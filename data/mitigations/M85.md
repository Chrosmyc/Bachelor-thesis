# Description
Conduct focused workshops for each subsystem where business analysts, application analysts, consultants, developers, and lead architects map legacy features to logical target components and identify dependencies.

# Prerequisites
Feature inventory and logical target architecture.

# Tools / Technologies
Gap analysis method; workshops

# Trade-offs / Side Effects
Requires coordination of multiple stakeholder groups.

# Effectiveness
The mapping approach was effective in identifying mappings and dependencies within high-level features.

# Stakeholders
- Business analysts; application analysts; consultants; developers; lead architects

# Contextual Constraints

- Migration requiring mapping from legacy features to target SOA components.


# Standardized Mitigation Entry
Focused gap-analysis workshops align legacy features with target components and expose dependencies that affect migration planning.

# Key Quote for Prompting
> "This mapping approach was effective"

# Logic Pattern
IF legacy features must be mapped to target architecture THEN APPLY focused gap-analysis workshops.
