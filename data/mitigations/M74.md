# Description
Restrict the number of programming languages used across microservices to support maintenance and reuse.

# Prerequisites
Organization has authority to define technology policies.

# Tools / Technologies


# Trade-offs / Side Effects
May reduce team-level technology freedom.

# Effectiveness
The cited practitioner study identifies language restriction as supporting maintenance and reuse.

# Stakeholders
- Architects; managers; developers; service owners

# Contextual Constraints

- Applies when teams select technologies for microservice implementation.


# Standardized Mitigation Entry
Limit programming-language diversity to reduce maintenance and reuse risks caused by uncontrolled technology heterogeneity.

# Key Quote for Prompting
> organizations should restrict the number of programming languages in order to facilitate maintenance and reuse

# Logic Pattern
IF technology flexibility could become uncontrolled THEN APPLY programming-language restrictions.
