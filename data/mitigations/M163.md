# Description
Restructuring and rewriting existing codebase to lower complexity without changing external behavior.

# Prerequisites
In-depth code analysis and a detailed architectural plan.

# Tools / Technologies
Static Analysis Tools, Automated Refactoring Frameworks

# Trade-offs / Side Effects
Requires substantial developer effort and initial time investment.

# Effectiveness
Effectively reduces technical debt and architecture clutter.

# Stakeholders
- Developers
- Architects

# Contextual Constraints

- Requires stable source code access.


# Standardized Mitigation Entry
Phased refactoring restructures complex codebases step-by-step to minimize dependencies and systematically repay technical debt.

# Key Quote for Prompting
> "Refactoring helps increase responsiveness and efficiency and reduce technical debt..."

IF [Technical debt threatens modernization stability] THEN APPLY [Phased Code Optimization (Refactoring)]
