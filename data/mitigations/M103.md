# Description
Structure software into independent components, each handling a specific business domain end-to-end and avoiding runtime dependencies on the monolith.

# Prerequisites
Business domains can be separated into independent components.

# Tools / Technologies
Self-Contained Systems architecture

# Trade-offs / Side Effects
SCSs are larger in scope than microservices.

# Effectiveness
Described as reducing interdependencies, limiting cascading failures, and enabling independent upgrade or scaling.

# Stakeholders
- Architecture team; development teams; operations teams; product owners

# Contextual Constraints

- Suitable when the system can be broken into distinct domains during modernization.


# Standardized Mitigation Entry
Self-Contained Systems mitigate interdependency and cascading-failure risks by creating autonomous domain-based components without runtime dependency on the monolith.

# Key Quote for Prompting
> "SCS is an architectural style that structures software into independent components, each handling a specific business domain end-to-end."

# Logic Pattern
IF a system can be split into business domains THEN APPLY Self-Contained Systems to reduce runtime interdependencies.
