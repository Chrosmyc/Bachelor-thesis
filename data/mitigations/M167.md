# Description
Authenticate and authorize every access request during modernization to reduce security risk.

# Prerequisites
Identity and access management capabilities; policy enforcement.

# Tools / Technologies
Zero trust security model

# Trade-offs / Side Effects
Requires mature access control and governance.

# Effectiveness
The paper gives a financial services firm as an example of adopting zero trust during modernization.

# Stakeholders
- Security teams; identity teams; compliance teams; users

# Contextual Constraints

- Applies especially in sensitive-data and regulated environments.


# Standardized Mitigation Entry
Zero trust reduces modernization security exposure by requiring every access request to be authenticated and authorized.

> 

# Logic Pattern
IF modernization expands access paths THEN APPLY zero trust authentication and authorization.
