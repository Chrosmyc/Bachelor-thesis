# Description
Choose and evaluate a suitable architecture, then prepare database design standards, reusable components, interface design standards, and coding standards.

# Prerequisites
Requirements specifications and knowledge of project nature.

# Tools / Technologies
Architecture styles; database design standards; interface standards; coding standards

# Trade-offs / Side Effects
Requires architectural evaluation and standardization effort.

# Effectiveness
Supports EO-III: detailed design document based on requirements specifications.

# Stakeholders
- Architects; design team; developers; QA team

# Contextual Constraints

- Applies in Architecture and Design stage.


# Standardized Mitigation Entry
Reduce architecture/design failure risk by selecting and evaluating architecture and enforcing design, interface, database, and coding standards.

# Key Quote for Prompting
> "Choose the suitable Architecture and define its components"

# Logic Pattern
IF architecture/design decisions are critical THEN APPLY architecture evaluation and design standards.
