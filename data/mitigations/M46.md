# Description
Use the staged model to distinguish evolution, servicing, phase-out, and close-down, and select technical solutions, processes, staff, and management activities appropriate to each stage.

# Prerequisites
The organization must assess the system’s current lifecycle stage.

# Tools / Technologies
Staged model, stage boundary analysis, planning models

# Trade-offs / Side Effects
May require empirical information and managerial discipline.

# Effectiveness
Proposed framework for planning; no quantified effectiveness reported.

# Stakeholders
- Managers
- architects
- maintainers
- business owners

# Contextual Constraints

- Useful when deciding between evolution
- servicing
- migration
- or retirement.


# Standardized Mitigation Entry
Stage-specific planning helps avoid applying inappropriate modernization strategies to systems in different lifecycle stages.

# Key Quote for Prompting
> Each stage has very different technical solutions, processes, staff needs and management activities.

# Logic Pattern
IF lifecycle stage is identified THEN apply stage-appropriate modernization processes/tools.
