# Description
Add a software seam, switch, routing mechanism, or API gateway that allows new components to be introduced transparently to consumers.

# Prerequisites
A routable interface or seam between consumers and legacy/new components.

# Tools / Technologies
Indirection layer; routing mechanism; API gateway

# Trade-offs / Side Effects
A more complex routing mechanism may be required depending on the system.

# Effectiveness
The layer enables traffic to move from old to new components without forcing immediate consumer disruption.

# Stakeholders
- Architects; developers; API consumers; operations

# Contextual Constraints

- Systems where consumer traffic can be routed or mediated.


# Standardized Mitigation Entry
Introduce an indirection layer to route consumers between legacy and new components during incremental migration.

# Key Quote for Prompting
> 

# Logic Pattern
IF new and old components must coexist THEN APPLY an indirection layer for transparent routing.
