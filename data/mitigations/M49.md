# Description
Raise the abstraction level used to express, reason about, and implement evolution; represent domain concepts explicitly.

# Prerequisites
Domain concepts and business objects must be identifiable.

# Tools / Technologies
Domain analysis, domain-specific languages, separation of business objects and control issues

# Trade-offs / Side Effects
Requires language/tool support and domain expertise.

# Effectiveness
Proposed as a promising research area.

# Stakeholders
- Architects
- domain experts
- developers

# Contextual Constraints

- Relevant during evolution and service-oriented modernization.


# Standardized Mitigation Entry
Higher abstraction mitigates modernization complexity by making domain concepts easier to reason about and change.

# Key Quote for Prompting
> Raise the level of abstraction in which evolution is expressed, reasoned about and implemented.

# Logic Pattern
IF modernization complexity is caused by low-level code changes THEN raise abstraction and represent domain concepts explicitly.
