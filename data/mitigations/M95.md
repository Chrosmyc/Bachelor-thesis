# Description
Begin security early in the lifecycle; use architectural review of application security measures, proven security strategies, code testing, code reviews, and vulnerability/penetration testing throughout modernization.

# Prerequisites
Security architect; security testing process; application access.

# Tools / Technologies
Code testing; code reviews; vulnerability testing; penetration testing.

# Trade-offs / Side Effects
Adds security work throughout lifecycle.

# Effectiveness
Proactively identifies and remediates security vulnerabilities.

# Stakeholders
- Security architect; developers; system architect; IT security

# Contextual Constraints

- All modernization programs
- especially digital/API/cloud changes.


# Standardized Mitigation Entry
Integrate security architecture review and vulnerability testing from the start to identify and remediate modernization security risks.

# Key Quote for Prompting
> "proactively identify and remediate security vulnerabilities"

# Logic Pattern
IF modernization changes or exposes applications THEN APPLY early security review and vulnerability testing.
