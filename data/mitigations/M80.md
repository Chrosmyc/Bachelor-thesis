# Description
Guide development so that each team owns one microservice and each microservice addresses a single, specific problem.

# Prerequisites
Team structuring and clear service ownership.

# Tools / Technologies


# Trade-offs / Side Effects


# Effectiveness
The paper indicates this principle supports focused microservice development.

# Stakeholders
- Development teams
- project managers
- software architects

# Contextual Constraints

- Applies during microservice development organization.


# Standardized Mitigation Entry
Assigning one microservice per team supports focused responsibility and helps each service address one specific problem.

# Key Quote for Prompting
> "microservice development should be guided by the principle of one microservice per team"

# Logic Pattern
IF microservices are being developed by multiple teams THEN APPLY one microservice per team ownership.
