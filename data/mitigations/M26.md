# Description
Automation should follow governance-driven planning that includes risk assessment, security evaluation, compliance review, and change management oversight.

# Prerequisites
Governance process; risk assessment capability; security evaluation; compliance review; change management oversight

# Tools / Technologies
Governance frameworks; COBIT; ISACA guidance

# Trade-offs / Side Effects


# Effectiveness
Aligns modernization with organizational objectives, risk tolerance, and operational continuity requirements

# Stakeholders
- Leadership; governance teams; technical teams; compliance teams; security teams; operational teams

# Contextual Constraints

- Organizations modernizing through automation
- cloud
- analytics
- AI
- or digital transformation initiatives


# Standardized Mitigation Entry
Governance-based automation reduces modernization risk by aligning automation with risk assessment, security, compliance, change control, and operational continuity.

# Key Quote for Prompting
> "Automation should follow governance-driven planning that includes risk assessment, security evaluation, compliance review, and change management oversight."

# Logic Pattern
IF automation is planned THEN APPLY governance-driven planning with risk, security, compliance, and change management oversight.
