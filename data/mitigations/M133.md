# Description
Use automated tools to analyze operating systems, middleware, and application frameworks against Azure support policies and flag unsupported, deprecated, or end-of-support components.

# Prerequisites
Access to environments and compatibility assessment tools.

# Tools / Technologies
Azure Migrate; other assessment tools

# Trade-offs / Side Effects


# Effectiveness
Detects compatibility issues across the environment without manual configuration reviews.

# Stakeholders
- Migration team
- application team
- operations team

# Contextual Constraints

- Applies to OS
- middleware
- and application framework compatibility against Azure support policies.


# Standardized Mitigation Entry
Automated compatibility analysis detects unsupported, deprecated, or end-of-support components before Azure migration.

# Key Quote for Prompting
> "flag components that are unsupported, deprecated, or approaching end of support"

# Logic Pattern
IF component compatibility is uncertain THEN APPLY automated compatibility analysis.
