# Description
Assess current/future business processes, capabilities, security, functional health, technical health, strategic alignment, financial impact, dependencies, and constraints; create a roadmap of application dispositions.

# Prerequisites
Access to current-state application and portfolio information.

# Tools / Technologies
Assessment tools; roadmap; portfolio analysis.

# Trade-offs / Side Effects
Requires upfront analysis effort before execution.

# Effectiveness
Provides data-driven insight and baseline scope to reduce risk.

# Stakeholders
- Enterprise architects; program manager; business owners; IT; finance

# Contextual Constraints

- First step of modernization planning.


# Standardized Mitigation Entry
Conduct an assessment and roadmap to select defensible modernization dispositions and reduce scope, dependency, and estimation risks.

# Key Quote for Prompting
> "The assessment provides detailed understanding of the current environment"

# Logic Pattern
IF modernization scope and dependencies are uncertain THEN APPLY application assessment and roadmap.
