# Description
Break monolithic legacy systems into loosely coupled independent services.

# Prerequisites
Architecture redesign

# Tools / Technologies


# Trade-offs / Side Effects
Increases architectural complexity and requires advanced management of distributed services.

# Effectiveness
High

# Stakeholders
- Software Architect
- Developer

# Contextual Constraints

- Relevant in distributed cloud deployments


# Standardized Mitigation Entry
Decoupling components improves scalability, maintainability, and fault isolation.

# Key Quote for Prompting
> “break up your application into independent services”

# Logic Pattern
IF system components are loosely coupled THEN scalability and fault tolerance improve
