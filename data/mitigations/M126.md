# Description
Fully redevelop a workload using cloud-native solutions when the legacy system is obsolete or modernization is not feasible, and reimagine the solution using Azure capabilities such as PaaS, automation, and AI.

# Prerequisites
Modernization feasibility has been assessed and found insufficient.

# Tools / Technologies
Azure PaaS, automation, AI, cloud-native solutions

# Trade-offs / Side Effects
Full redevelopment may require significant time and effort.

# Effectiveness
Avoids forcing modernization where rebuilding is more appropriate.

# Stakeholders
- Business owners
- architects
- technical leads
- execution teams.

# Contextual Constraints

- Applies when systems are obsolete
- too outdated
- too inflexible
- or modernization is not feasible.


# Standardized Mitigation Entry
Rebuilding mitigates infeasible modernization by replacing the workload with a new cloud-native solution.

# Key Quote for Prompting
> [Paraphrased evidence from G10] Rebuild is suitable when systems are obsolete or modernization is not feasible.

# Logic Pattern
IF modernization is not feasible THEN rebuild using a cloud-native solution.
