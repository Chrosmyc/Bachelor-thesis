# Description
Initially prioritize microservices that offer high value, minimal external dependencies, and minimal transition impact or risk.

# Prerequisites
Candidate microservices and dependency information.

# Tools / Technologies


# Trade-offs / Side Effects


# Effectiveness
The paper recommends this as the primary initial prioritization approach.

# Stakeholders
- Software architects
- development teams
- domain specialists

# Contextual Constraints

- Applies during decomposition and development sequencing.


# Standardized Mitigation Entry
Prioritizing high-value, low-dependency, low-impact candidates reduces transition risk during incremental modernization.

# Key Quote for Prompting
> "initially prioritize features with minimal impact or risk"

# Logic Pattern
IF multiple microservice candidates exist THEN APPLY prioritization by value, external dependencies, and transition impact/risk.
