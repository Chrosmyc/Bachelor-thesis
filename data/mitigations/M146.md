# Description
Prioritize capabilities, roadmap components, and migration projects using Time, Money, Risk, Reward, dependencies, complexity, cost-benefit, and stakeholder requirements.

# Prerequisites
Candidate capabilities, roadmap components, costs, benefits, risks, and dependencies are identified.

# Tools / Technologies
Portfolio Analysis: Risk, Reward, Time, and Money; Priority Quadrant; capability prioritization matrix

# Trade-offs / Side Effects
Requires consistent scoring and weighting across capabilities.

# Effectiveness
Provides a suggested prioritization framework and priority quadrant.

# Stakeholders
- Portfolio managers
- agency leadership
- business teams
- technology teams
- funding stakeholders

# Contextual Constraints

- Applies when multiple modernization components compete for time
- budget
- or resources.


# Standardized Mitigation Entry
Portfolio prioritization reduces roadmap risk by ranking modernization work using time, money, risk, reward, dependencies, and business value.

# Key Quote for Prompting
> "Portfolio Analysis: Risk, Reward, Time, and Money"

# Logic Pattern
IF multiple modernization options compete for limited resources THEN APPLY Time-Money-Risk-Reward prioritization.
