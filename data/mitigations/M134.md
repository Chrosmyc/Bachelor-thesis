# Description
Map how workload components communicate with each other and with internal organizational systems using network monitoring or application performance monitoring.

# Prerequisites
Access to runtime communication data.

# Tools / Technologies
Network monitoring tools; application performance monitoring

# Trade-offs / Side Effects


# Effectiveness
Helps determine grouping in migration waves.

# Stakeholders
- Migration planners
- architects
- operations team
- workload owners

# Contextual Constraints

- Applies when workloads have inter-component or internal system dependencies.


# Standardized Mitigation Entry
Internal dependency mapping supports dependency-aware migration wave grouping and reduces broken-dependency risk.

# Key Quote for Prompting
> "Use network monitoring tools or application performance monitoring"

# Logic Pattern
IF workload components communicate internally THEN APPLY internal dependency mapping.
