# Description
Combine gradual transition through the Strangler Pattern with autonomous and maintainable Self-Contained Systems.

# Prerequisites
Need for gradual modernization and separable business domains.

# Tools / Technologies
Strangler Pattern; Self-Contained Systems

# Trade-offs / Side Effects


# Effectiveness
Described as providing an optimal balance of incremental modernization and system independence.

# Stakeholders
- Architecture team; development teams; product owners; users

# Contextual Constraints

- Suitable when the system is not small/not mission critical enough for Big Bang and can be broken into distinct domains during modernization.


# Standardized Mitigation Entry
Combining Strangler Pattern with SCS mitigates migration risk while ensuring that new components are autonomous and maintainable.

# Key Quote for Prompting
> "The Strangler Pattern allows for a gradual transition, reducing risks associated with large-scale migrations, while SCS ensures that new components are autonomous and maintainable."

# Logic Pattern
IF modernization requires gradual transition AND the system can be divided into domains THEN APPLY Strangler Pattern with SCS.
