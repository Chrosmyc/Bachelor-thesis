# Description
Create a structured inventory of extracted rules, constraints, source locations, variables, effects, and confidence scores from the legacy artifacts.

# Prerequisites
Legacy source code, configurations, schemas, and available documentation must be accessible.

# Tools / Technologies
Business Rule Inventory, JSON, LLM-based extraction

# Trade-offs / Side Effects
Extraction may still miss deeply implicit rules; manual review may be needed.

# Effectiveness
The paper states BRI-to-BSG transfer was lossless in evaluated scenarios.

# Stakeholders
- Legacy analyzer
- domain experts
- developers
- auditors

# Contextual Constraints

- Applicable where source artifacts contain recoverable conditional logic and constraints.


# Standardized Mitigation Entry
A source-traceable Business Rule Inventory mitigates implicit-rule loss by documenting what was extracted, where it came from, and how confident the extraction is.

# Key Quote for Prompting
> "Each extracted rule is annotated with: a natural language description, the source location, input variables and output effects, and a confidence score."

# Logic Pattern
IF business rules are inventoried with source traceability THEN reviewers can verify modernization knowledge before code generation
