# Description
Assess new technologies from a maintenance and full-lifecycle perspective before adoption.

# Prerequisites
Technology options and expected maintenance implications must be evaluated.

# Tools / Technologies
Evaluation procedures, empirical studies, maintenance metrics

# Trade-offs / Side Effects
May slow adoption; evidence may be limited for emerging technologies.

# Effectiveness
Paper calls for evaluation procedures and warns new technologies can create maintenance problems.

# Stakeholders
- Architects
- managers
- maintainers
- researchers

# Contextual Constraints

- Applies before adopting OO
- components
- agents
- GUIs
- real-time/concurrent approaches
- etc.


# Standardized Mitigation Entry
Lifecycle evaluation mitigates the risk that modernization technologies create new maintenance problems.

# Key Quote for Prompting
> If such innovations are to be exploited successfully, the full lifecycle needs to be addressed, not just the initial development.

# Logic Pattern
IF a new technology is considered THEN evaluate its maintenance implications across the full lifecycle.
