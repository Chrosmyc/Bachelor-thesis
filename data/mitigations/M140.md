# Description
Start by identifying the boundaries of the existing system, application, or subsystem being replaced.

# Prerequisites
Access to system knowledge, architecture information, or SME input.

# Tools / Technologies
Architecture analysis

# Trade-offs / Side Effects


# Effectiveness
Boundary identification supports controlled incremental replacement.

# Stakeholders
- Architects; developers; SMEs

# Contextual Constraints

- Early modernization planning.


# Standardized Mitigation Entry
Identify system boundaries before migration to clarify what will be replaced and reduce dependency ambiguity.

# Logic Pattern
