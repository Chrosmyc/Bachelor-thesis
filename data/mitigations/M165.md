# Description
Start modernization with lower-risk areas such as automating manual processes or modernizing customer-facing applications before the core infrastructure.

# Prerequisites
Identification of lower-risk modernization candidates.

# Tools / Technologies
Workflow automation; customer-facing application modernization

# Trade-offs / Side Effects
Does not immediately resolve all core infrastructure risks.

# Effectiveness
The paper presents this as a common starting point for modernization.

# Stakeholders
- CIO; application owners; customer-facing teams; process owners

# Contextual Constraints

- Applies where non-core or lower-risk modernization targets exist.


# Standardized Mitigation Entry
Starting with low-hanging-fruit modernization creates early progress while limiting disruption to core infrastructure.

> 

# Logic Pattern
IF modernization risk is high THEN APPLY low-risk initial upgrades before core infrastructure changes
