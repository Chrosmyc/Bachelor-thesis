# Description
Gradually replace parts of the legacy system by introducing a proxy or intermediary layer that intercepts requests and redirects them to new components.

# Prerequisites
Ability to route selected functionality through an intermediary layer.

# Tools / Technologies
Proxy or intermediary layer

# Trade-offs / Side Effects
Domain separation may be unnecessary where the system is large but functionally cohesive.

# Effectiveness
Presented as ensuring continuity for users and avoiding disruption during gradual transition.

# Stakeholders
- Development team; architecture team; product owners; users

# Contextual Constraints

- Suitable when the system is tightly integrated
- cannot easily be split into distinct business domains
- and must remain operational.


# Standardized Mitigation Entry
The Strangler Pattern mitigates modernization transition risk by routing selected functionality to new components while the old system continues operating.

# Key Quote for Prompting
> "This approach enables teams to gradually transition functionality without disrupting existing public services, ensuring continuity for users."

# Logic Pattern
IF disruption must be avoided AND functionality can be redirected gradually THEN APPLY the Strangler Pattern.
