# Description
Defer modernization by retaining the application when the organization is not ready because of cost, dependencies, risks, or other factors.

# Prerequisites
Recognition that readiness barriers exist.

# Tools / Technologies


# Trade-offs / Side Effects
Modernization is postponed.

# Effectiveness
The guidance presents retain as the appropriate strategy under readiness constraints.

# Stakeholders
- Organization
- modernization decision-makers
- application owners

# Contextual Constraints

- Applies when readiness barriers make immediate modernization inappropriate.


# Standardized Mitigation Entry
Retain the application when readiness barriers indicate that modernization should be deferred until cost, dependency, and risk conditions are better understood or controlled.

# Key Quote for Prompting
> Source cue: retain when the organization is not ready due to cost, dependencies, risks, or other factors.

# Logic Pattern
IF readiness barriers exist THEN APPLY retain strategy.
