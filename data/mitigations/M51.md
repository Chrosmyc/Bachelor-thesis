# Description
Generalize migration beyond code to include data and objects as strategic assets.

# Prerequisites
Data and object assets must be identified and assessed.

# Tools / Technologies
Data migration planning, object migration, enterprise architecture

# Trade-offs / Side Effects
May increase migration scope and complexity.

# Effectiveness
Paper explicitly recommends generalizing migration to include data and objects.

# Stakeholders
- Data owners
- architects
- business owners
- developers

# Contextual Constraints

- Especially relevant where organizational data is more strategic than code.


# Standardized Mitigation Entry
Including data and objects mitigates the risk of code-only modernization missing key business assets.

# Key Quote for Prompting
> This certainly includes data and objects, not just code.

# Logic Pattern
IF migration is planned THEN include data and objects, not only code.
