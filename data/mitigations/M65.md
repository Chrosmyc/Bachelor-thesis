# Description
Allow a human reviewer or domain expert to inspect the BSG after specification generation and before code generation to catch extraction errors early.

# Prerequisites
BSG must be available in an inspectable form; domain experts or reviewers must understand the business rules.

# Tools / Technologies
BSG, Business Rule Inventory, source traceability

# Trade-offs / Side Effects
Requires human effort; not empirically evaluated as a full human-in-the-loop study in the paper.

# Effectiveness
The paper argues reviewers can inspect after Agent 2 and catch extraction errors before generation; future work proposes human-in-the-loop trust studies.

# Stakeholders
- Domain experts
- business owners
- architects
- auditors

# Contextual Constraints

- Relevant where rule correctness is safety-
- finance-
- or compliance-critical.


# Standardized Mitigation Entry
Human review of the BSG mitigates extraction and auditability risks by allowing domain experts to validate modernization knowledge before it becomes code.

# Key Quote for Prompting
> "With the BSG, a human reviewer can inspect after Agent 2 and catch extraction errors before any code is generated."

# Logic Pattern
IF a domain expert reviews the BSG before generation THEN incorrect or missing extracted rules may be caught early
