# Description
Analyze source code organization, existing features, dependencies, database structure, documentation, business characteristics, technologies, organizational aspects, behavior, and data flow before migration.

# Prerequisites
Stakeholder knowledge; logs; source code; design models; database diagrams; static and dynamic information; documentation.

# Tools / Technologies
Static and dynamic information of the system; database entity and relationship diagrams.

# Trade-offs / Side Effects


# Effectiveness


# Stakeholders
- Engineers responsible for migration; development teams; stakeholders

# Contextual Constraints

- Applies in the comprehension phase before architecture definition.


# Standardized Mitigation Entry
Perform a structured analysis of the legacy system before migration to provide the information needed for decomposition and architecture decisions.

# Key Quote for Prompting
> "The legacy system must be well-understood."

# Logic Pattern
IF migration planning begins THEN APPLY legacy-system understanding.
