# Description
Modernize workloads when legacy technologies, unsupported SDKs, or SaaS adoption require compatibility updates, and justify each effort with a clear business case.

# Prerequisites
Compatibility gaps and business case have been assessed.

# Tools / Technologies
SDKs, SaaS solutions

# Trade-offs / Side Effects
Compatibility modernization adds scope and should be justified.

# Effectiveness
Helps ensure the workload can support the selected migration or modernization path.

# Stakeholders
- Developers
- technical leads
- business owners
- execution teams.

# Contextual Constraints

- Applies when compatibility updates are required for target technology or SaaS adoption.


# Standardized Mitigation Entry
Required compatibility updates should be modernized with a clear business case to keep the target strategy viable.

# Key Quote for Prompting
> [Paraphrased evidence from G10] Compatibility-driven modernization should be justified with a clear business case.

# Logic Pattern
IF compatibility updates are required THEN modernize them with business-case justification.
