# Description
Build a clear inventory and understanding of workloads before deciding what each workload should become in the cloud.

# Prerequisites
Workloads have been identified and basic current-state information is available.

# Tools / Technologies
Workload inventory

# Trade-offs / Side Effects


# Effectiveness
Supports workload-specific strategy selection and reduces unsupported decision-making.

# Stakeholders
- Business owners
- technical leads
- cloud strategy team
- execution teams.

# Contextual Constraints

- Applies before selecting migration strategies for workloads.


# Standardized Mitigation Entry
A clear workload inventory helps the organization decide which migration strategy is appropriate for each workload and prevents unsupported strategy choices.

# Key Quote for Prompting
> [Paraphrased evidence from G10] A clear inventory and workload understanding should precede workload-specific cloud decisions.

# Logic Pattern
IF workload treatment decisions are needed THEN establish a clear inventory and workload understanding.
