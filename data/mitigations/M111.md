# Description
Treats the entire modernization program as a single engineered system with objectives, governed decision routines, acceptance criteria, and escalation pathways.

# Prerequisites
Program-level governance view and shared modernization objectives.

# Contextual Constraints

- Applies to modernization programs with multiple independently managed components.


# Standardized Mitigation Entry
Modernization-as-a-System Architecture mitigates fragmentation by governing the modernization program as one engineered system. It aligns teams around shared objectives, decisions, criteria, and escalation pathways.

# Key Quote for Prompting
> "Modernization-as-a-System Architecture treats the entire modernization program as a single engineered system."

# Logic Pattern
IF modernization spans multiple teams or components THEN APPLY Modernization-as-a-System Architecture.
