# Description
Attempt modernization during migration only when the team has adequate expertise and time; if readiness is lacking, defer modernization to a later phase.

# Prerequisites
Team readiness, skills, and timeline capacity are assessed.

# Tools / Technologies


# Trade-offs / Side Effects
Deferral can postpone modernization benefits.

# Effectiveness
Reduces risk and delays caused by readiness gaps.

# Stakeholders
- Execution teams
- technical leads
- business owners
- cloud strategy team.

# Contextual Constraints

- Applies when modernization requires skills or time beyond basic migration.


# Standardized Mitigation Entry
Modernization should be attempted only when skills and time are sufficient; otherwise, deferral reduces delivery risk.

# Key Quote for Prompting
> [Paraphrased evidence from G10] If the team lacks readiness, modernization should be deferred to a later phase.

# Logic Pattern
IF team skills or time are insufficient THEN defer modernization.
