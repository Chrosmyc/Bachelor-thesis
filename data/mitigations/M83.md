# Description
Create a catalogue for each business component showing reusability and functional dependencies with other components, enabling independent and parallel migration planning.

# Prerequisites
Identified candidate services and business components.

# Tools / Technologies
Component catalogue

# Trade-offs / Side Effects
Requires additional documentation and maintenance of component information.

# Effectiveness
The catalogue provided an overview of components whose migration could be performed independently and in parallel.

# Stakeholders
- Architecture Board; business architects; software architects; migration team

# Contextual Constraints

- Target architecture with multiple logical components and functional dependencies.


# Standardized Mitigation Entry
A business component catalogue captures reusability and dependencies, supporting safer sequencing and parallelization of modernization work.

# Key Quote for Prompting
> "provides an overview of components whose migration can be performed independently"

# Logic Pattern
IF target components have dependencies THEN APPLY a component catalogue to document reusability and coupling.
