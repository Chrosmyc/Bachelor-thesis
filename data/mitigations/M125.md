# Description
Avoid rehosting workloads with unresolved performance, reliability, architectural, or technical debt problems; modernize these workloads during migration to address root causes.

# Prerequisites
Workload issues and root causes have been identified.

# Tools / Technologies


# Trade-offs / Side Effects
Modernizing during migration can increase complexity and timeline risk.

# Effectiveness
Prevents carrying forward unresolved issues and reduces later rework.

# Stakeholders
- Technical leads
- execution teams
- operations teams
- business owners.

# Contextual Constraints

- Applies when rehost is considered for a problematic workload.


# Standardized Mitigation Entry
Problematic workloads should not be rehosted unchanged; root causes should be modernized to avoid carrying debt into Azure.

# Key Quote for Prompting
> [Paraphrased evidence from G10] Problematic workloads should be modernized during migration to address root causes.

# Logic Pattern
IF a workload is problematic THEN avoid plain rehost and modernize root causes.
