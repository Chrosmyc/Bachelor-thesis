# Description
Use the SMART-MP process and SMIG interview guide to gather migration context, legacy component information, candidate service information, and target SOA environment information; use answers to identify difficulty, effort, risks, and potential migration issues.

# Prerequisites
Stakeholders and technical personnel must be available for interviews and analysis.

# Tools / Technologies
SMIG; SMART Tool; Artifact Templates

# Trade-offs / Side Effects
Requires structured interviews and analysis effort before implementation.

# Effectiveness
The paper reports that SMART-MP analysis served as a blueprint in the MSS case and supported successful service development.

# Stakeholders
- Architects
- developers
- project managers
- service provider
- service consumers

# Contextual Constraints

- Applicable to early SOA migration planning and pilot selection.


# Standardized Mitigation Entry
SMART-MP and SMIG mitigate poorly grounded SOA migration by systematically eliciting context, risks, costs, and issues before implementation.

# Key Quote for Prompting
> "The use of this instrument assures broad coverage and consistent analysis of difficulty, risk, and cost issues."

# Logic Pattern
IF SOA migration risks are uncertain THEN APPLY SMART-MP/SMIG to elicit and structure migration issues.
