# Description
Use A/B deployments to expose increments of capabilities to smaller user sets and dark launches to deliver specific functionality increments.

# Prerequisites
Ability to release functionality incrementally to user subsets.

# Tools / Technologies
A/B deployments; dark launches.

# Trade-offs / Side Effects
Requires release management and monitoring capability.

# Effectiveness
Reported as successful in commercial and government sectors to improve user adoption.

# Stakeholders
- Deployment team; users; operations; product owners

# Contextual Constraints

- Incremental deployment scenarios.


# Standardized Mitigation Entry
Use A/B deployments or dark launches to reduce deployment shock and improve user adoption.

# Key Quote for Prompting
> "improve user adoption"

# Logic Pattern
IF deployment may affect many users THEN APPLY A/B deployment or dark launch techniques.
