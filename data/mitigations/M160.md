# Description
Apply local, small-scale changes through new technologies to address specific application issues while leaving the core architecture and technology unchanged; the paper presents it as low investment and less risky than bigger transformations.

# Prerequisites
Specific local application issue; unchanged core architecture and technology are acceptable; often suitable under tight budget.

# Tools / Technologies
New technologies for local changes

# Trade-offs / Side Effects
Can introduce cost from multiple changes, patchwork applications, bad design, and insufficient review.

# Effectiveness
The paper describes it as common and successful with comparatively low investment.

# Stakeholders
- Management; modernization team; application owners

# Contextual Constraints

- Suitable for current problems such as improving KPIs
- or when budget is tight in the middle of the year.


# Standardized Mitigation Entry
Use the duct tape approach for small local modernization problems under tight budget, recognizing that it can create architecture and governance side effects.

# Key Quote for Prompting
> "The duct tape approach for app modernization is ideal for small-scale changes with a greater return."

# Logic Pattern
IF the modernization need is local and budget is tight THEN apply small-scale duct tape changes rather than a larger transformation.
