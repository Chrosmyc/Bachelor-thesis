# Description
Create a central governing body with steering, core, program management, business change management, and architecture teams to manage the migration and align business, organizational, and technical decisions.

# Prerequisites
Stakeholder representation from senior management, business architects, software architects, technical managers, external consultants, and application developers.

# Tools / Technologies


# Trade-offs / Side Effects
Adds governance structure and coordination overhead.

# Effectiveness
The paper states that the committee fulfilled the need for a governing body and contributed toward successful migration.

# Stakeholders
- Senior management; business architects; software architects; technical managers; external consultants; application developers

# Contextual Constraints

- Large-scale
- long-term legacy-to-SOA migration.


# Standardized Mitigation Entry
A central Program Management Committee coordinates technical, organizational, and business aspects of migration and provides governance for a large-scale SOA modernization.

# Key Quote for Prompting
> "a central governing body with suitable governance of the entire migration process is indispensable"

# Logic Pattern
IF migration is large-scale and multifaceted THEN APPLY central governance through a Program Management Committee.
