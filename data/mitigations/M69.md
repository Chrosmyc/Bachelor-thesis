# Description
Ask developers to describe required code changes given service requirements, inputs/outputs, and target SOA characteristics; estimate effort and risk. Use code analysis tools or architecture reconstruction if needed.

# Prerequisites
Service requirements, target SOA information, and access to knowledgeable technical personnel or analysis artifacts.

# Tools / Technologies
Code analysis tools; architecture reconstruction

# Trade-offs / Side Effects
Depends on credibility of developer input or quality of analysis artifacts.

# Effectiveness
In the case, original developers were involved, input was credible, and no extra code/architecture analysis was needed.

# Stakeholders
- Developers
- architects
- project managers

# Contextual Constraints

- Applicable during gap analysis before strategy development.


# Standardized Mitigation Entry
Gap analysis mitigates hidden adaptation risk by identifying required changes and estimating effort before implementation.

# Key Quote for Prompting
> "The discussion of the changes that are necessary for each component is used as the input to calculate these preliminary estimates."

# Logic Pattern
IF required changes are uncertain THEN APPLY gap analysis and estimate effort/risk.
