# Description
Use white-box modernization and reverse engineering techniques to understand internal system structures, recover information from source code, and support the modernization process.

# Prerequisites
Access to source code and ability to analyze internal structures.

# Tools / Technologies
Reverse engineering tools

# Trade-offs / Side Effects
The paper does not specify trade-offs; it notes that many approaches rely on recovering information from source code.

# Effectiveness
The paper states that reverse engineering techniques are promising and have been used to understand systems and create tools supporting modernization; no quantified effectiveness is reported.

# Stakeholders
- Developers; architects; maintainers

# Contextual Constraints

- Applicable where modernization requires understanding internal structures or recovering information from source code.


# Standardized Mitigation Entry
White-box modernization and reverse engineering support modernization by recovering system information and improving understanding of internal structures.

# Key Quote for Prompting
> "reverse engineering techniques are promising and have been used to gain a grasp of the systems and to create tools to assist in the modernization process"

# Logic Pattern
IF internal system knowledge is required THEN APPLY white-box modernization and reverse engineering techniques.
