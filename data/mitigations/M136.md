# Description
Create a risk register to identify, assess, prioritize, and monitor potential risks affecting cloud adoption, including technical, operational, and organizational factors.

# Prerequisites
Defined workload scope and risk management process.

# Tools / Technologies
Risk register; governance documentation

# Trade-offs / Side Effects


# Effectiveness
Provides visibility into potential blockers and their value, enabling proactive risk management.

# Stakeholders
- Governance team
- migration team
- workload owners
- risk owners

# Contextual Constraints

- Applies across all workloads in cloud adoption.


# Standardized Mitigation Entry
A workload-level risk register makes cloud adoption risks visible, assessable, prioritizable, and monitorable.

# Key Quote for Prompting
> "identify, assess, prioritize, and monitor potential risks"

# Logic Pattern
IF cloud adoption risks must be managed THEN APPLY a risk register for all workloads.
