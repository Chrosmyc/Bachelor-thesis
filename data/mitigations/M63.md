# Description
Split modernization into specialized agents for legacy analysis, specification generation, code transformation, and equivalence validation instead of one monolithic prompt.

# Prerequisites
Legacy artifacts must be available and the pipeline must define clear artifact handoffs between agents.

# Tools / Technologies
LangGraph, OpenAI-compatible LLMs

# Trade-offs / Side Effects
More pipeline complexity than single-prompt generation; still insufficient without feedback.

# Effectiveness
Only full AgentModernize with feedback achieved non-zero mean BER across all tested backbones.

# Stakeholders
- Modernization architects
- developers
- QA teams
- domain experts

# Contextual Constraints

- Applicable to LLM-based modernization workflows where behavioral preservation is required.


# Standardized Mitigation Entry
Decomposing modernization into extraction, specification, transformation, and validation reduces the risk of monolithic LLM failure by assigning separate responsibilities to specialized agents.

# Key Quote for Prompting
> "AgentModernize decomposes legacy modernization into four agent-handled phases."

# Logic Pattern
IF modernization tasks are separated into specialized agents THEN monolithic prompt failure risk is reduced
