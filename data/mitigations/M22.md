# Description
Leave legacy code in place during intermediate migration phases and discard it only after all functionality has been modernized.

# Prerequisites
Ability to coexist with legacy functionality temporarily.

# Tools / Technologies


# Trade-offs / Side Effects
Increases coexistence complexity during the intermediate phases.

# Effectiveness
Reduces instability caused by premature code removal.

# Stakeholders
- Software Architect
- Developer

# Contextual Constraints

- Applies during transaction-based migration.


# Standardized Mitigation Entry
Retaining legacy code during intermediate modernization phases can reduce destabilization risks.

# Key Quote for Prompting
> "Legacy code can just be left in place."

# Logic Pattern
IF legacy functionality remains operational during migration THEN instability risks decrease.
