# Description
Only migrate to a new framework, such as Microsoft .NET Framework to .NET Core, when there is strong business justification.

# Prerequisites
Assessment of business justification for framework migration.

# Tools / Technologies
Business justification analysis; framework migration planning

# Trade-offs / Side Effects
Avoiding a framework change may defer some modernization work.

# Effectiveness
Avoids significant development effort and testing when no strong justification exists.

# Stakeholders
- Business owners
- architects
- developers
- migration team

# Contextual Constraints

- Applies when framework migration is being considered.


# Standardized Mitigation Entry
Avoiding framework changes without strong business justification reduces avoidable development and testing effort during modernization.

# Key Quote for Prompting
> "Only migrate to a new framework ... if there's a strong business justification"

# Logic Pattern
IF a framework change lacks strong business justification THEN AVOID unnecessary framework migration.
